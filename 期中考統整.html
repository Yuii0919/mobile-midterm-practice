Ch1 行動裝置應用程式開發

  • Android：基於 Linux 核心與開源元件的行動 OS，由 Google 與開放手機聯盟持續發展。
  • AndroidManifest.xml（app/src/main/）：記錄 Activity、Service、套件名、版本與權限等。
  • Gradle：自動化建構與依賴管理。

Manifest 常考元素

元素／屬性意義
<manifest>根節點，包住整個 App 宣告
xmlns:androidAndroid 命名空間，供屬性使用
package應用程式套件名稱
<application>圖示、名稱、主題等應用層設定

講義 OCR 曾出現「intnet-filter」，正式標籤為 intent-filter。

程式與資源目錄

  • java（或 kotlin 目錄）：主要程式碼；Activity 常繼承 AppCompatActivity，以 setContentView(...) 載入版面。
  • res：drawable（圖/XML）、layout（XML 版面）、mipmap（不同解析度圖示）、values（顏色、字串、主題）。

建立專案與 Gradle（常考名詞）

  • New Project 常見欄位：Name、Package name、Save location、Language（Kotlin）、Minimum SDK。
  • Minimum SDK 越低 → 可安裝的舊裝置越多，但專案能直接使用的新 API 越少（需做相容或降功能）。
  • Gradle：建構與依賴管理；常見檔案含專案／模組的 build.gradle（或 .kts）、settings.gradle。
  • Project 檢視：實際磁碟資料夾結構；Android 檢視：以開發角度整理（常考「兩者差異」）。

Ch2 介面設計與佈局

  • Layout：用 XML 描述 UI 排列，檔案放在 res/layout/（例如 activity_main.xml）。
  • 常見佈局：LinearLayout、RelativeLayout（以相對位置排列）、FrameLayout（堆疊、常作根容器或單一子視圖）、ConstraintLayout（Studio 預設常用）。

LinearLayout

  • 依序排列子視圖；android:orientation：vertical 或 horizontal，未設時預設 horizontal。

寬高（必考）

值說明
wrap_content依內容（文字、圖、子元件）撐開
match_parent盡量填滿父容器給的空間
0dp在 ConstraintLayout 中常配合 constraint 撐滿
固定 dp固定像素邏輯尺寸，少用於需適配多螢幕時

ConstraintLayout

  • 元件間以約束（constraint）定位，無固定「巢狀階層」思維。
  • 每個元件至少要有一個水平 + 一個垂直約束，否則 XML 會報錯。

Margin 與 Padding

  • Margin：元件外側與其他元件／邊界的距離。
  • Padding：元件內側內容與邊框的距離。
  • 可細分 Top/Bottom/Start/End、Horizontal/Vertical、Left/Right（padding 同理）。

元件常考屬性
TextViewid、text、textColor、textSize（sp）、background
Buttontext、textAllCaps、backgroundTint、enabled
EditTexthint、inputType、maxLines
RadioGrouporientation、checkedButton（XML 預設選中 id）

Selector 與 Shape

  • Selector：依狀態（按下、選取等）切換 drawable／顏色，多放在 res/drawable/。
  • Shape：rectangle、oval、line；corners 圓角、stroke 邊框、solid 填色。
標籤重點屬性用途
<selector>（根節點）狀態選擇器根節點
<item>android:state_pressed 等符合狀態時套用哪個 drawable／色
<item>android:drawable該分支要顯示的圖或 shape
  • Corners：可用 android:radius 四角同圓角，或 topLeftRadius、topRightRadius、bottomLeftRadius、bottomRightRadius 分開設定。

gravity 與 layout_gravity

  • android:gravity：控制自己內部子內容的對齊。
  • android:layout_gravity：此 View 在父容器中的對齊（center、left、top…）。

資源與單位

  • res/values/colors.xml、strings.xml：集中管理顏色與字串（利於多語系與維護）。
  • 另可放 dimens.xml 統一間距與尺寸常數（與講義「values 內變數值」觀念一致）。
  • dp：與螢幕密度無關的「尺寸」單位（寬高、margin、padding）。
  • sp：文字大小；會隨系統「字型大小」設定縮放，故 TextView 字級用 sp。
  • dpi：螢幕物理密度概念；dp 與實際像素換算會依 dpi 而變，考題常問「為何用 dp 而非 px」。

Ch3 物件控制與事件監聽

  • Kotlin 檔 .kt（如 MainActivity.kt）：處理互動、更新 UI、生命週期等。
  • AppCompatActivity：AndroidX，可在較舊 OS 使用較新 UI 能力。
  • onCreate：Activity 建立時呼叫；內呼叫 super.onCreate(...) 後 setContentView(R.layout.activity_main) 載入版面。

連結 XML 元件

val tvName = findViewById<TextView>(R.id.tvName)
tvName.text = "你的名字"
  • 更新文字：textView.text = "..."
  • 以程式設字級：tv.setTextSize(TypedValue.COMPLEX_UNIT_SP, 30f)（30f 為 Float）。

事件監聽

  • 使用者操作稱為 event；按鈕常用 setOnClickListener。
  • 兩種寫法：Lambda 區塊 setOnClickListener { ... } 或傳入監聽物件（講義所謂「函式形式」）。

資源 ID 與範例邏輯

  • 編譯器產生 R 類別：R.layout.xxx、R.id.xxx、R.string.xxx、R.drawable.xxx、R.color.xxx。
  • XML 元件若要在程式控制，必須有 android:id="@+id/名稱"。
  • RadioButton：執行期可用 isChecked 判斷；搭配 when 對應剪刀／石頭／布等文字或邏輯。
  • 猜拳亂數：(Math.random() * 3).toInt() 得到 0、1、2（講義範例常對應剪刀／石頭／布）。
  • 換行字元在字串中為 \n。

Ch4 訊息提示元件（課程延伸）

複習講義主軸至 Kotlin；此章對照練習題與實作，建議熟記 API 名稱與情境。

  • Toast：Toast.makeText(context, "訊息", Toast.LENGTH_SHORT).show()；第三參數亦可用 LENGTH_LONG。一定要 show()。
  • 自訂版面：建立 layout（如 custom_toast.xml），inflate 後設給 Toast 的 view；可用 setGravity 調位置與偏移。
  • Snackbar：需錨定 View（如 root）；可 Snackbar.make(...).setAction("動作") { }.show()；時長含 LENGTH_INDEFINITE 等。
  • AlertDialog：AlertDialog.Builder(context) → setTitle、setMessage、setItems（清單）、setSingleChoiceItems（單選）；按鈕為 setPositiveButton／setNegativeButton／setNeutralButton。
  • 情境：Toast 短提示；Snackbar 可附「復原」等動作；AlertDialog 適合確認、選一項、中斷流程的互動。

Kotlin 基礎（講義後半）

  • 設計目標：較 Java 精簡、強化空值安全；亦為 Android 官方推薦語言之一。
  • val：唯讀，不可重新指派；var：可變。
  • 數值型別含 Int、Long、Float、Double 等。

Null safety（必考觀念）

  • 預設變數不可為 null；未初始化會編譯錯誤。
  • 需表達「沒有分數」vs「0 分」時，不能一律用 0 混為「缺考」。
  • 可空型別在型別後加 ?，例如 Int?。
  • ?.：安全呼叫，左側為 null 時整段為 null，不拋例外。
  • !!：非空斷言，保證不為 null；若執行時為 null → NullPointerException。安全性：?. 優於 !!。

類型推論與字串

  • Kotlin 為靜態型別；有初值時編譯器可推斷型別（如 val s = "Kotlin" → String）。
  • 推論後型別固定，不能對該變數呼叫不相容方法（講義例：languageName.inc() 無法編譯）。
  • 字串模板："Hello, $name"、"總和：${a + b}"；可搭配 playground 練習（講義連結如 pl.kotl.in）。

類別、條件、函式

  • class 大寫開頭；屬性用 val／var；方法用 fun。
  • if-else 可作運算式指派給變數，例如 val msg = if (n > 0) "正" else "非正"。
  • when：可寫 when { 條件 -> 值; else -> 預設 }；比巢狀 if 易讀。
  • fun 名稱(參數): 回傳型 { ... }；可有預設參數，例如 fun greet(x: Int = 10)、多參數各給預設值。
  • 單一運算式函式可寫成 fun sq(n: Int) = n * n，省略大括號與 return。
  • 匿名函式：無名稱，可賦值給變數或當參數傳遞。
  • 可空型別常用 Elvis：score ?: 0 表示左側為 null 時改用右側（與「缺考無分數」語意搭配考）。
  • 每個可執行檔案頂層僅一個 fun main() 作為程式進入點（考「不能多個 main」）。
