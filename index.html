第一週與 Android 基礎

  • 建立專案欄位必背：Name、Package、Save location、Language、Minimum SDK。
  • AndroidManifest.xml 是 App 設定核心：宣告 activity、啟動畫面、權限、主題。
  • res/layout 放 UI XML；res/drawable 放圖片與 shape；res/values 放字串/顏色/主題。
  • strings.xml 統一管理文字，方便多語系與維護；colors.xml 統一顏色命名。
  • Project 檢視看實際資料夾，Android 檢視看開發導向分類（考題常問兩者差異）。
  • Minimum SDK 越低相容設備越多，但可直接用的新 API 越少。
  • Gradle 是建構與依賴管理工具，常見檔：build.gradle、settings.gradle。
  • 上機考關鍵：流程熟悉、題目先易後難、留最後 3-5 分鐘檢查。

Kotlin 基礎

  • val（唯讀）不可重新指派；var（可變）可重新指派。
  • 型別推論：可寫 val name = "Kotlin"，編譯器仍固定型別為 String。
  • 可空型別必加 ?，例如 Int?；未加 ? 不能直接塞 null。
  • 安全呼叫 ?.：空值時不崩潰；非空斷言 !!：若為 null 會直接 NPE。
  • 字串模板常考：$name（變數）、${a+b}（運算式）。
  • 條件：if-else 可回傳值；多分支用 when 較清楚。
  • 函式：fun 名稱(參數:型別): 回傳型別，可設定預設參數。
  • 單一表達式函式可簡寫，省略大括號與 return。
  • 考題陷阱：0 與 null 意義不同（例如 0 分 vs 缺考）。

Ch2 介面設計與佈局

  • 常見佈局：LinearLayout（線性）、FrameLayout（堆疊）、ConstraintLayout（約束）。
  • LinearLayout 未寫 orientation 預設為 horizontal（常考）。
  • ConstraintLayout 每個元件至少要有水平與垂直方向約束，否則會警告/錯誤。
  • 寬高 4 種概念：wrap_content、match_parent、0dp(match_constraint)、固定尺寸。
  • 在 ConstraintLayout 內常用 0dp 搭配左右約束做滿版寬度。
  • layout_margin* 是外距；padding* 是內距（常混淆）。
  • gravity 控內容；layout_gravity 控元件在父層的位置。
  • 文字大小用 sp，元件尺寸/間距用 dp。
  • RadioGroup 管理多顆 RadioButton，確保同時只選一個。

Ch3 物件控制與事件監聽

  • 程式綁定順序：setContentView() 先載入畫面，再 findViewById() 綁元件。
  • XML 元件必須有 id，Kotlin 才能用 R.id.xxx 控制。
  • 事件監聽主力：setOnClickListener { ... }，通常寫在 onCreate()。
  • 文字更新：TextView.text = "內容"；換行符號是 \\n。
  • 字體大小動態設定：setTextSize(TypedValue.COMPLEX_UNIT_SP, 30f)。
  • RadioButton 判斷常用 isChecked，搭配 when 轉成對應文字。
  • 猜拳範例：電腦亂數 (Math.random()*3).toInt() 會得到 0/1/2。
  • 依課堂對應：0=剪刀、1=石頭、2=布，再比對玩家出拳判斷勝負。
  • R 類別常考：R.id、R.layout、R.string、R.drawable、R.color。

Ch4 訊息提示元件

  • Toast 基本語法：Toast.makeText(context, msg, duration).show()。
  • Toast 3 參數：Context、訊息字串、持續時間（LENGTH_SHORT/LENGTH_LONG）。
  • 自訂 Toast 流程：先做 custom_toast.xml，再指定給 Toast 的 view。
  • setGravity(Gravity.TOP/BOTTOM/CENTER, x, y) 控制出現位置與偏移。
  • Snackbar 優勢：可附互動按鈕，語法 setAction("按鈕") { ... }。
  • Snackbar 時長：LENGTH_SHORT、LENGTH_LONG、LENGTH_INDEFINITE。
  • AlertDialog 常見 API：setTitle、setMessage、setItems、setSingleChoiceItems。
  • 按鈕型 AlertDialog：setPositiveButton / setNegativeButton / setNeutralButton。
  • 考點：Toast 偏提示；Snackbar 可回應；AlertDialog 最適合要使用者做選擇/確認。
