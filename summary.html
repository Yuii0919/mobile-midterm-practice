第一週與 Android 基礎

  • Android Studio 建立專案常見欄位：Name、Package、Minimum SDK、Language。
  • AndroidManifest.xml 用來宣告 Activity、權限、主題與 app 重要資訊。
  • res 內常見資料夾：layout、drawable、values。
  • Gradle 負責專案建構與依賴管理。

Kotlin 基礎

  • val 不可重新指派、var 可重新指派。
  • 型別推論可省略型別宣告，但仍是靜態型別語言。
  • Null safety：可空型別加 ?，安全呼叫用 ?.，!! 可能拋出 NPE。
  • 條件可用 if-else 或 when。
  • 函式以 fun 宣告，可含參數、回傳值與預設值。

Ch2 介面設計與佈局

  • 常見佈局：LinearLayout、FrameLayout、ConstraintLayout。
  • LinearLayout 未設定 orientation 預設為 horizontal。
  • ConstraintLayout 元件至少要有水平與垂直方向約束。
  • 字體大小建議用 sp，尺寸與間距建議用 dp。
  • gravity 是內容對齊，layout_gravity 是元件對父層對齊。

Ch3 物件控制與事件監聽

  • 先 setContentView(...) 才能正確 findViewById(...)。
  • 按鈕監聽常用 setOnClickListener { ... }。
  • 讀取輸入：editText.text，更新畫面：textView.text = ...。
  • RadioButton 透過 isChecked 判斷選中項目。
  • R 類別存取所有資源 ID。

Ch4 訊息提示元件

  • Toast：Toast.makeText(context, msg, duration).show()。
  • Snackbar 可加動作按鈕：setAction("按鈕") { ... }。
  • AlertDialog 常見方法：setTitle、setMessage、setItems、setSingleChoiceItems。
  • AlertDialog 適合需要確認、單選、多選等互動情境。
